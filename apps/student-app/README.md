# student-app — 学生端（uni-app）

## 定位

本工程是医小管的学生端，使用 uni-app 开发，一套代码支持 H5 和微信小程序两种发布目标。

## 技术栈

- uni-app + TypeScript
- uView UI（uni-app 组件库）
- Pinia（状态管理）
- WebSocket（接收教师实时消息、主动推送）
- SSE（AI 流式输出）

## 发布目标

| 阶段 | 发布形式 | 说明 |
|---|---|---|
| 一期 | H5（网页） | 可挂公众号菜单，学生直接访问 |
| 一期后期 | 微信小程序 | 同一套代码，换编译目标发布 |
| 二期 | 教师轻审批小程序 | 复用部分组件，独立入口 |

## 页面范围（一期）

- 登录页（账号密码 / 预留微信登录入口）
- 首页（AI 对话入口、事务导办入口、常用入口聚合）
- AI 对话页（核心页面）
  - 气泡消息流（区分 AI / 教师 / 学生身份）
  - 流式输出效果
  - 快捷链接卡片（意图匹配推荐）
  - 快捷提问词
  - 呼叫老师按钮
- 事务申请页（空教室申请表单）
- 申请进度查询
- 个人主页（学生描述、历史问答）
