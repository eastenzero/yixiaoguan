## 四、教务系统对接技术评估

### 阻塞项: 验证码 (CAPTCHA)
教务系统登录必须提交 `j_captcha`，无法绕过。可选方案：

| 方案 | 可行性 | 说明 |
|------|--------|------|
| A. 人工登录截图 | ✅ 即时可行 | 你手动登录，截图/复制关键页面内容 |
| B. OCR 验证码识别 | ⚠️ 中等 | 需训练或接入验证码识别服务 |
| C. 移动端 API 探查 | ⚠️ 值得尝试 | 企业微信"学生课表"可能走不同 API，验证码要求可能更低 |
| D. 联系教务处获取文档 | ✅ 推荐 | 直接获取规章制度 PDF/Word，免去抓取 |

### 对接开发参考
- 教务系统基础 URL: `https://jwc.sdfmu.edu.cn/academic/`
- 密码传输: 前端 AES-ECB 加密后 POST
- Session: Cookie-based (JSESSIONID)
- 防护: CAPTCHA + CSRF Token + X-Frame-Options DENY
