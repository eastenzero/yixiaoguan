## 审查维度

### 维度一：frontmatter 完整性校验
每个文件必须包含以下字段且格式合法：
- `title`: 非空字符串
- `category`: 必须是以下枚举之一：入学与学籍 / 奖助贷补 / 就业与毕业 / 心理与测评 / 竞赛与第二课堂 / 证件与校园服务 / 事务申请与审批 / 学院通知与常见流程 / 生活服务 / 教务与课程 / 信息化服务 / 图书馆 / 学校概况 / 招生与深造
- `tags`: 非空数组，至少1个标签
- `source`: 非空字符串，包含 "kimi/report-" 引用
- `campus`: 必须是 "泰安" / "济南" / "通用" 之一
- `status`: 值为 "draft"

### 维度二：内容质量校验
- **标准答复**: `## 标准答复` 节存在且内容 ≥ 50 字
- **无占位符**: 正文中不得含有 "xxx"、"[待补充]"、"[TBD]" 等未填充内容
- **无幻觉信息**: 检查是否有明显编造的电话号码（非以下格式之一：0531-xxxxx、0538-xxxxx、1xxxxxxxxxx）

### 维度三：编号与命名
- 文件名必须是 `KB-20260409-NNNN.md`（NNNN 四位补零）
- 编号必须从 0001 连续到 0084，无跳号、无重号

### 维度四：内容去重
检查以下重点去重对：
- 0031（网上报修）vs KB-0151：是否互补而非重复
- 0034（人脸采集）vs 0068（人脸采集）：两者是否实质重复（若重复请标注）
- 0003（济南电话）vs 0004（泰安电话）：是否已成功分校区
