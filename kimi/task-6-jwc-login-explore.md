# 任务：教务管理系统登录探查

## 身份
你是一个数据侦察员，需要通过编程方式登录山东第一医科大学教务管理系统，探查其内部功能结构和可用数据。

## 目标
编写并执行 Python 脚本，登录教务系统，映射其菜单结构和功能页面，提取对"学生校园助手知识库"有用的信息。

## 登录信息
- **教务系统 URL**: https://jwc.sdfmu.edu.cn/academic/common/security/affairLogin.jsp
- **账号**: [已清理]
- **密码**: [已清理]
- ⚠️ 凭据已在探查完成后清理

## 执行步骤

### Step 1: 分析登录页面
- 先用 requests 获取登录页面 HTML，分析表单结构（action URL、字段名、隐藏字段、CSRF token 等）
- 检查是否有验证码

### Step 2: 模拟登录
- 使用 Python requests.Session() 模拟登录
- 保持 session cookies
- 验证登录是否成功（检查响应内容/状态码/重定向）

### Step 3: 映射菜单结构
- 登录成功后，获取主页面
- 提取所有导航菜单、链接、功能模块
- 记录每个模块的名称和 URL

### Step 4: 探查关键页面（每个页面只访问一次，避免触发安全机制）
重点关注以下页面：
1. 培养方案 / 教学计划
2. 选课相关页面（选课规则、时间安排）
3. 成绩查询页面结构
4. 学籍管理（学籍异动、休学、复学、转专业）
5. 考试安排
6. 重修/补考相关
7. 毕业要求 / 学分要求
8. 教学评价

### Step 5: 提取规章制度类内容
如果发现规章制度、管理办法、操作指南等静态内容页面，提取其文本内容。

## 技术要求
- 使用 Python 3 + requests 库
- 如果 requests 未安装，先 pip install requests
- 设置合理的 User-Agent（模拟浏览器）
- 请求间隔至少 2 秒，避免被封
- 总请求数控制在 30 次以内
- 如果登录失败或遇到验证码，立即停止并记录原因
