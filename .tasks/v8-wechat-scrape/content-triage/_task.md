# CONTENT-TRIAGE — 文章内容分类与价值评估

**所属**: v8-wechat-scrape / PHASE 3
**优先级**: P1
**执行者**: Kimi
**预估工时**: 2h
**依赖**: FILTER-DOWNLOAD

## 任务目标

对已下载的文章进行二次分类，判断每篇文章能转化为几条 KB 以及转化方式。

## 分类标准

**Tier-1（直接转 KB）**: 包含具体流程/规定/指南/联系方式，信息密度高
- 示例: "2025年秋季学期选课通知" → 选课时间+系统操作流程
- 示例: "图书馆暑假开放时间调整" → 具体开放时段表
- 示例: "奖学金评审办法" → 评审条件+申请流程

**Tier-2（提取部分内容）**: 文章整体是新闻/活动，但含有可用的事实片段
- 示例: 新生入学须知（长文）→ 仅提取报到流程/材料清单
- 示例: 开学典礼通知（含报到安排）→ 仅提取住宿报到时间

**Tier-3（跳过）**: 纯活动回顾/领导讲话/科研新闻/无实用价值

## 执行步骤

### Step 1: 自动化预分类

对 `wechat-exports/` 下每篇文章，读取标题+前 300 字，输出初步 Tier 判断。

可批量输出格式（每行一条）：
```
{公众号名称} | {文件名} | Tier-{1/2/3} | {预估KB条数} | {关键信息点}
```

### Step 2: 人工核查抽样（T1）

对 Tier-1 文章随机抽取 10% 确认分类合理性。

### Step 3: 统计汇总

各账号 Tier 分布、预估 KB 条目总数。

## 输出

`kimi/wechat-content-triage.md`:

```markdown
## 分类汇总

| 账号 | Tier-1 | Tier-2 | Tier-3 | 预估KB条数 |
|------|--------|--------|--------|-----------|

## Tier-1 文章清单（完整）
| 账号 | 文件名 | 预估KB条数 | 关键信息点 |
...

## Tier-2 文章清单（可提取部分）
...
```

## 验收标准

- AC-TRI-01: 所有下载文章均有 Tier 标注
- AC-TRI-02: Tier-1 文章记录关键信息点
- AC-TRI-03: 预估总 KB 条目数 ≥ 60
