# SYNC-ARTICLES — 同步 A/B 类公众号文章列表

**所属**: v8-wechat-scrape / PHASE 2
**优先级**: P0
**执行者**: Kimi
**预估工时**: 2h
**依赖**: DISCOVER (fakeid 已确认)

## 任务目标

对 A 类（8个）和 B 类（8个）公众号，通过 API 分页同步全部文章列表，记录文章总数。

## 执行步骤

### Step 1: 逐个同步文章列表（翻页）

```python
# 示例：同步单个公众号的全部文章（每页20条）
GET /api/public/v1/article?fakeid={fakeid}&begin=0&size=20   # 第1页
GET /api/public/v1/article?fakeid={fakeid}&begin=20&size=20  # 第2页
# 直到返回文章数 < 20（末页）
```

也可通过 Web UI 点击【同步】按钮自动完成，更直观。

**同步顺序**（按优先级）:
1. A01 教务部
2. A02 研究生处
3. A03 学工
4. A04 后勤
5. A05 心理健康
6. A06 图书馆
7. A07 就业
8. A08 科研部
9. B01~B08（依次）

### Step 2: 记录各账号文章总数

每个账号同步完成后，记录：
- 公众号名称
- fakeid
- 文章总数
- 最早/最新文章日期

## 输出

`kimi/wechat-sync-stats.md`：

| 账号 | 等级 | fakeid | 文章总数 | 最新文章日期 | 状态 |
|------|------|--------|---------|------------|------|

## 注意事项

- 微信接口频率限制：翻页之间建议间隔 1-2 秒
- 工具会自动缓存文章列表，重复调用不耗费额外配额
- 大号（教务部/官方主号）可能有数千篇文章，同步需要时间

## 验收标准

- AC-SYN-01: A 类 8 个账号文章列表全部同步完成
- AC-SYN-02: B 类至少 5 个账号同步完成
- AC-SYN-03: 统计表包含所有已同步账号的文章数量
