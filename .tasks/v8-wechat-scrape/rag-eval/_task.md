# RAG-EVAL-V8 — RAG 评测

**所属**: v8-wechat-scrape / PHASE 4
**优先级**: P1
**执行者**: Kimi
**预估工时**: 1h
**依赖**: INGEST-V8

## 任务目标

入库完成后，扩充评测集并验证 RAG 质量不退化。

## 新增测试题

根据实际下载到的文章主题，添加 5-10 个新测试问题：

**建议题型（根据实际 KB 内容选择）：**
- 公众号发现的新通知/政策（如"2025年度奖学金评审什么时候开始？"）
- 后勤服务信息（如"宿舍热水供应时间是几点？"）
- 教务通知（如"本学期期末考试什么时候？"）
- 图书馆服务（如"黄河图书馆自习室怎么预约？"）
- 就业服务（如"学校就业指导中心在哪里？"）

**注意**: 题目需基于实际入库的 KB 内容，不能提前预设。

## 回归基线

v7 最终结果：Recall@5=86.0%, 拒答=92.9%

## 执行步骤

1. 根据实际入库的公众号来源 KB，编写 5-10 个新测试题
2. 追加到 `scripts/eval/eval-set-v3.yaml`（生成 v4 或在 v3 基础上追加）
3. 通过 SSH 运行 `scripts/eval/run_eval_v3.py`
4. 输出评测报告

## 输出

- `scripts/eval/eval-set-v4.yaml`（包含 v3 全部题目 + v8 新增 5-10 题）
- `scripts/eval/eval-report-v4.md`
- `kimi/rag-eval-v8-report.md`（摘要）

## 验收标准

- AC-EVAL8-01: 评测集新增 5-10 题，覆盖公众号来源内容
- AC-EVAL8-02: Recall@5 ≥ 85%（允许较 v7 的 86.0% 略降，因题目增多）
- AC-EVAL8-03: 原有题目 Recall 不退化
- AC-EVAL8-04: 拒答准确率 ≥ 90%
