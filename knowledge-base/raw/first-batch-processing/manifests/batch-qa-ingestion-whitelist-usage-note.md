# Batch QA Ingestion Whitelist 使用说明

> **面向对象**：执行入库操作的同学  
> **必读顺序**：先读本文档 → 再检查 `batch-qa-manual-review-list.csv` → 最后操作 `batch-qa-ingestion-whitelist.csv`

---

## 一、Whitelist 定义（可入库候选清单）

`batch-qa-ingestion-whitelist.csv` 记录的是**可入库候选清单**，包含**已知存在非结构性质量问题**的材料。

### 1.1 什么是 "可入库"
- ✅ **可以**进入知识库入库流程
- ✅ **不阻断**自动化入库管道
- ⚠️ **已知存在**质量缺陷，需后续逐步优化

### 1.2 什么是 "非结构性问题"
| 问题类型 | 说明 | 是否阻断入库 |
|----------|------|--------------|
| 规则条目重复 | 内容与其他条目重复，但非P0级别 | ❌ 否 |
| 证据状态不完整 | 缺少部分佐证材料，但非指定类型 | ❌ 否 |
| 模板类材料 | 格式为模板/示例，非正式文档 | ❌ 否 |
| 时效性待确认 | 过期或待确认有效性，但非P0 | ❌ 否 |
| 待人工复核 | 标记需复核但非指定类型 | ❌ 否 |

### 1.3 关键字段说明
| 字段 | 含义 |
|------|------|
| `kb_id` | 知识库条目编号 |
| `material_id` | 材料唯一标识 |
| `source_batch` | 来源批次（BATCH-A1/A2/A3） |
| `reason` | 进入whitelist的具体原因 |
| `ready_for_ingest` | **固定为 TRUE**，表示可入库 |

## 二、Manual-Review 定义（人工审核清单）

`batch-qa-manual-review-list.csv` 记录的是**必须人工干预**后才能决定入库与否的材料。

### 2.1 什么情况下必须进 Manual Review
满足**任一**条件即进入：

| 条件 | 严重程度 | 典型场景 |
|------|----------|----------|
| severity 包含 P0 | P0-严重 | KB编号冲突 |
| issue_type 为指定类型 | P1-告警 | 源文件缺失、版本冲突 |

### 2.2 Manual Review 清单中的问题类型
| 问题类型 | 条目数 | 严重等级 | 必须人工原因 |
|----------|--------|----------|--------------|
| KB编号冲突 | 21条 | P0-严重 | 跨批次编号重复，需重新分配 |
| 源文件缺失 | 9条 | P1-告警 | A3批次源文件待补充 |
| 版本冲突 | 4条 | P1-告警 | 同一文档多版本，需确认保留哪个 |

### 2.3 关键字段说明
| 字段 | 含义 |
|------|------|
| `kb_id` | 知识库条目编号 |
| `material_id` | 材料唯一标识 |
| `source_batch` | 来源批次 |
| `issue_type` | 问题类型（KB编号冲突/源文件缺失/版本冲突） |
| `severity` | 严重程度（P0-严重 / P1-告警） |
| `required_action` | 建议的人工处理动作 |

## 三、两者边界（Whitelist vs Manual Review）

```
┌─────────────────────────────────────────────────────────────┐
│                    QA 检查结果                               │
│                                                             │
│   ┌─────────────────────┐    ┌─────────────────────────┐   │
│   │   Manual Review     │    │      Whitelist          │   │
│   │   (34条 - 56.7%)    │    │   (26条 - 43.3%)        │   │
│   │                     │    │                         │   │
│   │  • KB编号冲突        │    │  • 规则条目重复         │   │
│   │  • 源文件缺失        │    │  • 证据状态不完整       │   │
│   │  • 版本冲突          │    │  • 模板类材料           │   │
│   │                     │    │  • 时效性待确认         │   │
│   │  【阻塞性/结构性问题】│    │  【非阻塞性质量问题】    │   │
│   │                     │    │                         │   │
│   │  ❌ 不可直接入库     │    │  ✅ 可入库后优化        │   │
│   └─────────────────────┘    └─────────────────────────┘   │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 3.1 判定规则（唯一标准）

```
IF (severity 包含 "P0") 
   OR (issue_type ∈ ["KB编号冲突", "源文件缺失", "版本冲突"])
THEN → Manual Review
ELSE IF (无重复冲突)
THEN → Whitelist
```

### 3.2 典型边界案例

| 场景 | 归属 | 原因 |
|------|------|------|
| KB-0061 与 KB-0082 编号重复 | Manual Review | P0-严重，结构性冲突 |
| RULE-A1-010 与 RULE-A1-011 内容重复 | Whitelist | 非P0，非结构性冲突 |
| 源文件标记为 "待补充" | Manual Review | 属于源文件缺失类型 |
| 证据状态标记为 "不完整" | Whitelist | 非P0且非指定类型 |

## 四、执行人员注意事项

### ⚠️ 关键流程：先看 Manual 再入库

```
正确流程：
   步骤1: 检查 batch-qa-manual-review-list.csv
          ↓ 确认本条目不在这34条中
   步骤2: 检查 batch-qa-ingestion-whitelist.csv
          ↓ 确认条目在其中且 ready_for_ingest=TRUE
   步骤3: 执行入库操作
```

### 4.1 入库前检查清单
- [ ] 确认 `kb_id` 不在 `batch-qa-manual-review-list.csv` 中
- [ ] 确认 `kb_id` 在 `batch-qa-ingestion-whitelist.csv` 中
- [ ] 确认 `ready_for_ingest` 字段值为 `TRUE`
- [ ] 阅读 `reason` 字段了解已知问题

### 4.2 禁止操作
| 禁止行为 | 原因 |
|----------|------|
| ❌ 直接入库 Manual Review 清单中的条目 | 存在结构性问题，可能破坏知识库完整性 |
| ❌ 修改 whitelist 中 `ready_for_ingest` 的值 | 该字段由 QA 流程统一管控 |
| ❌ 擅自移动条目 between 两个清单 | 归属由判定规则决定，非人工指定 |

### 4.3 发现问题如何处理
若发现条目归属有疑问：
1. **不要**擅自修改CSV文件
2. **不要**擅自执行/阻止入库
3. 联系 QA 收口工程师复核判定规则
