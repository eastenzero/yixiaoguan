# 医小管一期功能清单

## 1. 一期做什么

### 1.1 学生端

- 智能问答
- 事务导办
- 空教室申请
- 申请进度查询
- 常用官网入口聚合

### 1.2 教师端

- 今日学生提问列表
- 高频问题汇总
- 待人工处理问题
- 空教室审批
- 人工答复转知识库

### 1.3 管理端

- 知识条目审核
- 标签与分类管理
- 用户角色管理
- 系统配置
- 审计日志

### 1.4 知识库建设

- 建立业务问题域分类
- 建立知识条目模板
- 建立原始材料与知识条目的映射关系
- 先人工整理高频和稳定材料

## 2. 一期不做什么

- 原生 App
- 微服务拆分
- 心理预警
- 用户画像
- 推荐系统
- 社区或内容互动
- 多模态问答
- 统一身份认证正式对接
- 教务、学工、官网正式系统对接
- 大规模向量引擎升级

## 3. 一期依赖什么

进入开发前必须先具备以下前置条件：

- 开发总文档完成
- 知识库整理规范完成
- 一期业务边界冻结
- 核心流程定稿
- 角色与权限边界明确
- 首批知识条目样本形成

## 4. 功能依赖关系

学生问答依赖：

- 已整理的知识条目
- 问答与会话模型
- AI 服务接入规范

事务导办依赖：

- 分类清晰的办事流程知识
- 常用入口信息

空教室申请依赖：

- 申请单模型
- 审批流程模型
- 状态通知机制

教师工作台依赖：

- 问答记录
- 人工处理状态
- 高频问题归并逻辑

人工答复转知识库依赖：

- 知识条目模板
- 审核流
- 来源追溯规则

## 5. 一期完成标准

满足以下条件，可视为一期闭环成立：

- 学生可以完成提问并获得回答
- 未解决问题可以转入教师处理
- 教师处理结果可以转为知识条目草稿
- 管理员可审核并发布知识
- 学生可以发起空教室申请并查询状态
- 教师可以审批并留痕
