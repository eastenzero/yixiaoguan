# 项目定位与建设目标

## 1. 项目定位

医小管定位为面向高校学生事务与辅导员场景的智能服务平台，核心能力不是单纯生成回答，而是：

- 为学生提供稳定、可追溯的事务咨询与办事引导
- 为教师提供问题分流、人工处理与审批工作台
- 为管理侧提供知识治理、权限控制与运营支撑

项目目标是从“比赛作品”转向“可上线、可交接、可演进”的正式系统。

## 2. 解决的问题

当前校园学生事务场景存在几个典型问题：

- 高频问题重复咨询，占用辅导员时间
- 通知、流程、模板、手册分散，学生难以快速找到答案
- 学生事务往往停留在“查通知”，没有真正形成申请和审批闭环
- 教师人工答疑缺乏统一沉淀，无法反哺知识库

医小管要解决的不是单点问答，而是“咨询、导办、申请、审批、沉淀”的完整链路。

## 3. 三类核心用户

学生：

- 提问
- 查看办事说明
- 发起事务申请
- 查询处理进度

教师/辅导员：

- 查看和处理学生问题
- 处理审批事项
- 统一人工答疑
- 将标准答复沉淀为知识

管理员/运营：

- 审核知识
- 管理分类和标签
- 管理角色和权限
- 维护配置与审计记录

## 4. 建设目标

一期目标：

- 建立问答闭环
- 建立申请审批闭环
- 建立教师答疑转知识的闭环

中期目标：

- 扩展到更多学生事务场景
- 提升知识运营与多端体验
- 建立更稳定的消息和后台管理能力

长期目标：

- 与校内系统逐步打通
- 支持多学院扩展
- 抽象通用模块与垂直模块

## 5. 边界与约束

当前阶段明确约束如下：

- 不以旧代码兼容为目标
- 不一开始进入复杂微服务
- 不把心理预警、画像、推荐等高扩展能力提前拉入近期建设
- 不把正式对接校内系统排在基础闭环之前
