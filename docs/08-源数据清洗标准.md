# 源数据清洗标准

## 1. 目的

本文用于规范 `数据库部分材料/` 的首轮整理工作，目标不是“搬文件”，而是为后续格式转换、模型抽取和知识条目生产提供统一标准。

适用范围：

- `数据库部分材料/学生手册.md`
- `数据库部分材料/数据库部分材料/2025年工作/`
- `数据库部分材料/数据库部分材料/模板/`

## 2. 工作原则

- 保留原始文件，不在首轮整理中移动、重命名或覆盖原件。
- 先盘点和标记，再做格式转换和知识抽取。
- 先判断材料是否值得入库，再决定是否投入模型处理。
- 一切知识条目必须可追溯到原始来源路径。
- 优先整理高频、稳定、可复用的问题，不优先整理名单、汇总表和宣传材料。

## 3. 清洗对象层级

源数据整理分为四层：

1. 原始文件层：现有 `doc/docx/pdf/xls/xlsx/jpg/png/zip/rar/md`
2. 材料总表层：记录每份材料的元信息、分类判断和处理动作
3. 转换文本层：将可读材料转为 `md/txt/csv`
4. 知识条目层：按知识模板整理后的标准答复

首轮工作只覆盖前两层，并为第三层留出规范。

## 4. 文件类型处理规则

### 4.1 `doc` / `docx` / `pdf`

默认视为主要规则来源，优先检查以下内容：

- 是否包含明确的通知、制度、流程、条件、材料要求、时限
- 是否属于现行政策或仍可参考的办事规则
- 是否只是模板、名单附页或过期工作安排

处理规则：

- 有稳定规则价值：标记为 `可转知识`
- 仅含阶段性安排：标记为 `仅作时效通知`
- 仅含附件模板：标记为 `模板附件`
- 与现行规则冲突或明显过期：标记为 `过期待归档`

### 4.2 `xls` / `xlsx`

默认不直接作为问答主知识，而作为辅助材料检查：

- 是否包含字段说明、申报条件、评审规则、时间节点
- 是否只是名单、统计表、汇总表、登录编号、结果表

处理规则：

- 含规则说明页或字段规范：可提炼规则部分
- 纯名单、纯统计、纯结果：保留为附件，不进入主知识
- 申报模板、汇总模板：标记为 `模板附件`

### 4.3 `jpg` / `png`

默认不作为主知识来源，主要用于：

- 截图佐证
- 流程配图
- OCR 候选

处理规则：

- 与核心流程强相关且无其他文字版：标记为 `待OCR`
- 仅作说明截图：标记为 `图片附件`

### 4.4 `zip` / `rar`

压缩包只视为容器，不直接入知识流程。

处理规则：

- 标记为 `待解包`
- 解包后按内部实际文件类型重新分类

## 5. 材料价值分级

为提高效率，所有材料先做价值分级：

### A 级：优先入库

满足至少两项：

- 学生会高频咨询
- 规则较稳定
- 可以直接提炼为 FAQ 或流程
- 覆盖面广

典型示例：

- 入学与学籍流程
- 奖助贷补规则
- 毕业离校流程
- 心理测评登录与操作
- 证件补办流程

### B 级：可提炼，延后处理

特点：

- 有规则信息，但混杂大量附件、名单或阶段性内容
- 需要人工判断后再抽取

典型示例：

- 通知加附件打包材料
- 含模板、汇总表的奖学金评审包
- 大型活动审批相关材料

### C 级：只保留附件

特点：

- 不回答学生常见问题
- 主要是名单、统计、结果、汇总、宣传

典型示例：

- 获奖名单
- 人数统计表
- 优秀学生宣传稿
- 学生信息汇总

### D 级：暂缓

特点：

- 内容难读
- 明显过期
- 重复度高
- 当前项目阶段价值低

## 6. 分类映射规则

原始目录按年份和工作批次归档，不适合直接作为知识分类。知识分类统一映射到以下目录：

- `入学与学籍`
- `奖助贷补`
- `就业与毕业`
- `心理与测评`
- `证件与校园服务`
- `竞赛与第二课堂`
- `事务申请与审批`
- `学院通知与常见流程`

映射规则：

- 以学生提问场景为准，不以文件存放目录为准
- 同一文件可支持多个条目，但材料总表只保留一个主分类
- 如涉及多个场景，用 `tags` 处理交叉检索

## 7. 时效性判断规则

每份材料都要判断时效状态：

- `现行`：当前仍适用，或短期内仍应作为有效依据
- `阶段性`：仅对某一学年、批次、活动周期有效
- `待确认`：无法从文件名或内容中确认是否仍有效
- `过期`：明确被新规则替代，或时间窗口已结束且无复用价值

判断优先级：

1. 文件正文中的发布日期、执行时间、适用学年
2. 文件标题中的年份、届别、学期
3. 所在目录的时间线信息
4. 如仍不明确，标记为 `待确认`

## 8. 去重与版本处理规则

以下情况视为重复或近重复：

- 同名文件不同格式，如 `doc` 和 `pdf`
- 同一事项的修订稿、终稿、附件包
- 文件名不同但正文高度一致

处理规则：

- 保留原件，不删除
- 在材料总表中指定一个 `主参考文件`
- 其余标记为 `重复参考`、`历史版本` 或 `附件包`

## 9. 模板识别规则

以下内容默认优先识别为模板，而不是知识条目：

- 标题含“模板”“汇总表”“附件”“登记表”“申请表”“委托书”
- 内容主要是空表格、填写样例、固定格式证明
- 目的是填报或打印，不是解释规则

处理规则：

- 模板进入附件管理
- 如模板附带填写规则，可只抽取规则部分入知识条目

## 10. 输出要求

首轮清洗后，每份材料至少需要产出以下判断：

- 主分类
- 材料类型
- 时效状态
- 适用对象
- 价值分级
- 是否模板
- 是否重复
- 建议处理动作

## 11. 首批优先范围

首批优先处理以下内容：

1. `学生手册.md` 中学工办事流程相关章节
2. `2025下-25级新生工作` 中与入学、学籍、奖助、心理测评直接相关的通知
3. `2025上-21级毕业工作` 中毕业、就业、离校、档案相关材料
4. `模板/` 中与奖助贷补、证件补办、第二课堂直接相关的规则性模板

暂不优先：

- 大批量名单和统计表
- 宣传稿、总结稿、典型事迹
- 纯成果展示类材料

## 12. 与后续阶段的衔接

完成本标准对应的清洗后，再进入下一阶段：

1. 批量格式转换
2. 模型抽取结构化信息
3. 生成知识条目草稿
4. 人工复核发布
