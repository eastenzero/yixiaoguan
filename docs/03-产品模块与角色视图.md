# 产品模块与角色视图

## 1. 角色视图

学生视角关注四件事：

- 问问题
- 找流程
- 提申请
- 查进度

教师视角关注四件事：

- 看提问
- 处理提问
- 审批事项
- 沉淀知识

管理视角关注四件事：

- 审核知识
- 维护分类
- 管理权限
- 查看日志与配置

## 2. 核心模块

用户与权限模块：

- 登录认证
- 角色管理
- RBAC 权限控制
- 用户资料
- 审计日志

问答与会话模块：

- 学生提问
- AI 回答
- 会话记录
- 高频问题归并
- 未解决问题上报

知识库模块：

- 知识条目管理
- 标签与分类
- 来源记录
- 版本管理
- 审核发布
- 下线归档

申请审批模块：

- 空教室申请
- 审批流转
- 审批意见
- 状态通知
- 操作留痕

导办与入口聚合模块：

- 办事入口
- 办理说明
- 链接聚合
- 跳转前置说明

教师工作台模块：

- 今日提问
- 高频问题
- 待处理问题
- 待审批事项
- 一键转知识库
- 数据统计

通知与消息模块：

- 站内消息
- 审批通知
- 申请结果通知
- 风险提示

## 3. 端侧定位

教师端：

- 主阵地为 Web
- 适合处理高信息密度、复杂审批和管理操作

学生端：

- 主阵地为 H5 与小程序
- 强调移动端高频访问和低门槛入口

管理端：

- 可并入教师端后台体系
- 侧重知识治理与权限管理

## 4. 核心业务闭环

闭环 A：问答与知识增长

- 学生提问
- AI 回答
- 未解决问题进入教师处理
- 教师标准答复转知识草稿
- 管理员审核发布
- 后续问答命中新知识

闭环 B：事务申请与审批

- 学生发起申请
- 教师审批
- 学生接收通知
- 学生查询进度
- 系统全流程留痕

## 5. 当前优先级

近期优先围绕两条闭环设计产品，不扩散到社区、推荐、画像、多模态等能力。
