## 二、服务边界与职责

### 2.1 business-api（主业务后端）

| 职责 | 说明 |
|---|---|
| 认证鉴权 | JWT 签发/校验，RBAC 权限拦截 |
| 用户管理 | 批量导入、激活、角色分配 |
| 问答会话 | 创建会话、存储消息、管理会话状态 |
| 教师介入 | WebSocket 推送、会话状态切换 |
| 知识库管理 | 条目 CRUD、审核状态机、标签管理 |
| 申请审批 | 空教室申请创建、审批流转 |
| 通知推送 | 站内消息、手动群发推送 |
| 审计日志 | 自动记录所有关键操作 |
| AI 代理 | 转发前端请求到 ai-service，不直接调大模型 |

**关键原则：business-api 不直接调用 DashScope API。所有 AI 能力通过 HTTP 调用 ai-service。**

### 2.2 ai-service（AI 能力层）

| 职责 | 说明 |
|---|---|
| RAG 问答 | 接收问题 → 检索 ChromaDB → 组装 Prompt → 调用 DashScope → 返回流式回答 |
| 链接匹配 | 用问题 Embedding 匹配 quick_links 集合，返回相关链接 |
| 知识草稿 | 根据教师回复自动生成知识条目草稿 |
| 相似聚类 | 对高频问题做向量聚类归并 |
| 向量入库 | 知识条目/快捷链接审核通过后，向量化写入 ChromaDB |

**关键原则：ai-service 不处理认证、不操作 PostgreSQL。纯粹做 AI 计算。**

## 五、端口与路由规划

| Redis | 6379 | **不对外暴露** | 仅服务间内网 |

## 七、安全要点（一期最低标准）

| 项 | 措施 |
|---|---|
| 认证 | JWT + BCrypt 密码加密，Token 有效期 2 小时 |
| 传输 | HTTPS（Let's Encrypt 免费证书） |
| 权限 | RBAC，菜单+按钮级权限拦截 |
| API 限流 | Redis 限流（防止 DashScope API 被刷） |
| SQL 注入 | MyBatis-Plus 参数化查询 |
| 审计 | 所有关键操作写入 yx_audit_log |
| 密钥管理 | `.secrets/.env` 不提交 Git |
