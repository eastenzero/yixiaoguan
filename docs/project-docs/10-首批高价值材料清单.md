# 首批高价值材料清单

## 1. 目的

本文不是完整材料总表，而是首批执行清单，用于在进入工具链设计前先锁定“最值得处理的一批材料”。

对应索引文件：

- [knowledge-base/raw/first-batch-material-index.csv](C:/Users/Administrator/Documents/code/yixiaoguan/knowledge-base/raw/first-batch-material-index.csv)

## 2. 范围统计

首批范围内已确认的文件规模如下：

- `学生手册.md`：1 份
- `2025下-25级新生工作/`：136 份
- `2025上-21级毕业工作/`：22 份
- `模板/`：106 份

合计扫描范围：265 份。

考虑到首轮目标是“搭建工具链并验证流程”，不建议一次性全量处理 265 份文件，而是先聚焦高价值样本。

## 3. 选取标准

首批样本按以下标准筛选：

- 能直接回答学生高频问题
- 能形成流程型或 FAQ 型知识条目
- 与现有分类目录直接对应
- 兼顾“制度依据”和“学生操作说明”
- 包含至少一组可以做版本校验的平行材料

## 5. 为什么这一步是必要的

工具链选择必须围绕材料结构来定，而不是抽象讨论。

只有先锁定一批高价值样本，后面这些决策才有依据：

- 哪些格式转换是必须做的
- 哪些材料适合直接让模型抽取
- 哪些材料需要先做模板/附件剥离
- 哪些目录更适合作为第一轮验证场景

## 6. 下一步如何衔接工具链

基于这份清单，后续工具链可以按三段来定：

1. 扫描与索引
   目标：从首批样本生成完整材料总表和文件元信息。

2. 格式转换
   目标：将首批高价值 `doc/docx/pdf` 转成模型可处理的统一文本层。

3. 模型抽取
   目标：从转换后的文本里提取“适用对象、条件、流程、材料、时限、注意事项”，再生成知识条目草稿。

## 7. 结论

在真正敲定工具之前，先做这份首批材料锁定是必要步骤，而且已经完成。

下一步就可以基于这份清单来定工具链，不再需要对整个原始材料目录做泛泛讨论。
