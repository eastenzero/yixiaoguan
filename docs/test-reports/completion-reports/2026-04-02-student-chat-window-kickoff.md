# 学生端 AI 对话模块窗口纪要（2026-04-02）

## 1. 背景与目标

当前学生移动端开发聚焦 `AI 对话`，这是项目核心亮点。

本轮窗口确认的优先目标：

1. 提升对话页视觉质量（布局层次、组件风格、参考稿一致性）
2. 完成来源引用闭环（AI 回复底部显示参考资料，标题可点击）
3. 快捷问题支持一键发送（已具备基础能力，后续优化交互体验）

暂缓项：

- 根据回复内容智能推荐相关链接/入口（受知识库参数约束，后续再做）

---

## 2. 现状核查结论

### 2.1 已实现能力

- 流式输出：`/api/chat/stream` + 前端 `fetch + ReadableStream` 已存在
- 快捷问题：空状态和会话态均有预设问题点击发送
- 来源引用 UI：AI 消息下方已能展示“参考资料”列表

### 2.2 当前缺口

- 来源点击目前依赖 `source.url`，但 AI 服务返回的 `sources` 仅包含：
  - `entry_id`
  - `title`
  - `content`
  - `score`
- 学生端当前无知识详情页路由，无法直接承接来源跳转
- 对话页视觉风格与参考稿（`_references/前端风格/学生移动端/_1/screen.png`）仍有差距

---

## 3. 本窗口决策（已确认）

来源引用跳转策略采用 **混合方案**：

1. 优先跳转知识详情页（按 `entry_id`）
2. 若跳转失败或详情拉取失败，则降级为弹层展示引用摘要
3. 若存在外部 `url`，保留外链打开/复制能力

---

## 4. 计划落地范围

### 4.1 功能闭环

- 为来源引用增加统一点击处理：
  - 内链优先（知识详情）
  - 失败降级（弹层）
  - 外链兜底（H5 打开 / 非 H5 复制）
- 新增学生端知识详情页面与 API 封装（最小可用）

### 4.2 体验优化

- 对话页来源区视觉可读性优化（标题、编号、可点击反馈）
- 对话页关键组件样式细化（遵循现有设计系统与参考稿气质）
