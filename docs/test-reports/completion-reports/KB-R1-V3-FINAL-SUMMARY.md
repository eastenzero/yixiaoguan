| A1 处理 | 保留不动 | 需重建（无实物）| v3 发现缺失 |

### 4.2 为什么不能直接执行 v2

若按 v2 执行，将产生以下问题：

```
v2 执行结果（假设）：
- KB-0061.md (实际A3内容) → KB-0082.md (编号归为A2)
  结果：编号是A2，内容还是A3 → ❌ 归属错误

- KB-0062.md (实际A3内容) → KB-0083.md (编号归为A2)
  结果：编号是A2，内容还是A3 → ❌ 归属错误

... 以此类推，0061~0079 全部归属错误
```

**后果**：
- 队列 CSV 同步后形成"编号正确、内容错误"的隐性污染
- 后续入库、检索、展示都会出现批次归属混乱
- 难以追溯和修复

### 4.3 v3 核心修正点

1. **归属判定标准化**：基于 frontmatter 的 material_id，而非文件名编号
2. **实物与队列对齐**：确保重命名后的编号与目标批次队列的 material_id 一致
3. **缺失项显式化**：A1 的 7 个条目无实物，明确标记为 need_rebuild
4. **REX 项隔离**：规则类条目禁止重命名，单独标记为 manual_review
