## 一、报告目的

### 1.1 背景
在 BATCH-QA 流程完成后，产生了两份清单：
- `batch-qa-ingestion-whitelist.csv` (26条)
- `batch-qa-manual-review-list.csv` (34条)

执行同学反馈："whitelist" 一词容易引起误解，可能被理解为 "完全没问题" 的材料清单。

### 1.2 目标
本文档说明**为何不改CSV文件名，而选择补充语义文档**来澄清概念。

## 二、为何不改CSV只补语义文档

### 2.1 不改CSV的三重原因

| 原因 | 详细说明 |
|------|----------|
| **系统耦合** | `batch-qa-ingestion-whitelist.csv` 文件名已被下游入库系统硬编码，改名会导致管道断裂 |
| **历史追溯** | 该CSV已在多个交付物中被引用，改名会破坏版本追溯 |
| **成本权衡** | 改名成本 > 语义澄清成本，且后者可复用于后续批次 |

### 2.2 补充文档的优势

```
┌────────────────────────────────────────────────────────────┐
│                     解决方案对比                            │
├────────────────────────────────────────────────────────────┤
│ 方案A: 修改CSV文件名                                        │
│   • 需改动: 文件名 + 所有引用 + 下游系统配置                 │
│   • 风险: 高（可能破坏自动化管道）                           │
│   • 收益: 低（仅解决命名问题）                               │
│   • 决策: ❌ 不采纳                                         │
├────────────────────────────────────────────────────────────┤
│ 方案B: 补充语义文档 (已采纳)                                 │
│   • 需改动: 新增 2 个说明文档                                │
│   • 风险: 无                                                │
│   • 收益: 高（可复用、可维护、可扩展）                        │
│   • 决策: ✅ 采纳                                           │
└────────────────────────────────────────────────────────────┘
```

## 三、Whitelist 语义澄清

### 3.1 原名称的歧义
- **字面理解**："白名单" = 完全没问题、免检通过
- **实际含义**："可入库候选清单" = 已知有问题，但非阻塞性

### 3.2 澄清后的定义

```
WHITELIST (本批次QA语境) =
    "可入库候选清单，包含已知非结构性质量问题"
    
    其中 "非结构性" 指：
    • 不涉及 KB 编号冲突
    • 不涉及 源文件缺失
    • 不涉及 版本冲突
    • 不涉及 P0 级别问题
```

### 3.3 与常规术语的对照

| 术语 | 常规理解 | 本批次QA实际含义 | 差异 |
|------|----------|------------------|------|
| Whitelist | 完全合格 | 有条件通过，已知有非阻塞问题 | 需额外说明 |
| Manual Review | 待人工确认 | 有结构性问题，必须人工修复后入库 | 一致 |
| Ready for Ingest | 准备就绪 | 可进入入库流程，但需知晓已知缺陷 | 需额外说明 |

## 四、两个CSV文件的语义定位

### 4.1 文件职责对比

| 维度 | Whitelist CSV | Manual Review CSV |
|------|---------------|-------------------|
| **语义定位** | 可入库候选池 | 阻塞性问题池 |
| **执行动作** | 可以入库 | 暂停，等待人工 |
| **问题性质** | 非结构性质量缺陷 | 结构性/配置性问题 |
| **严重程度** | P1-告警 / P2-提示 | P0-严重 / P1-告警 |
| **后续优化** | 入库后逐步修复 | 入库前必须解决 |
