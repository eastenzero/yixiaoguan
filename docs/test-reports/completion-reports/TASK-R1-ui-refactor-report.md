## 5. 新发现错误模式
1. 💡 **错误模式分析**：在进行样式更新时（例如调整基于状态展现不同颜色的 Badge 面板），极易陷入“修改并重新组装业务 script 中用于返回颜色配置的函数”的窘境，这是对“不可变更业务处理逻辑”严格指引的一种擦边破坏。
   - **最佳实践**：面对必须按业务映射状态配色的视觉重构，应保持底层状态输出代码不动，将数据映射转为单纯的 `<template>` 选择器匹配（如类绑定匹配）并利用 CSS 实体属性解决，达成表现与逻辑纯净解耦。
   - *（建议评估是否补录进 `ai-antipatterns.md` 作为强约束性指引依据。）*
