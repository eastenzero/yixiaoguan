## 五、为什么 v2 不能直接执行

v2 方案的核心假设是：**保留 A2、重建 A3**，即：
- 0061~0079 视为 A2 内容，rename 到 0082~0100
- 0080~0081 视为 A2 内容，rename 到 0101~0102

**但实际验证发现：**
- 0061~0079 的 material_id 与 **A3 队列** 匹配（MAT-20260324-0050~0061, MAT-CP-001~008）
- 只有 0080~0081 的 material_id 与 A2 队列匹配

**如果执行 v2：**
- 会把 A3 内容（0061~0079）错误地重命名为 A2 编号（0082~0100）
- 导致编号与内容严重错位
- 后续队列字段更新后会产生"看起来一致、实际错位"的隐性错误
- 这种错误比"文件不存在"更危险，因为它更不容易被发现

## 六、v3 相比 v2 的核心修正点

| 维度 | v2 方案 | v3 方案 |
|-----|--------|--------|
| **判定依据** | 基于 old_kb_id 推断归属 | 基于 material_id 核实归属 |
| **0061~0079归属** | 错误地认为是 A2 | 正确地识别为 A3 |
| **0080~0081归属** | 正确地认为是 A2 | 正确地识别为 A2 |
| **A1内容** | 未考虑 | 明确标记为缺失需重建 |
| **REX规则项** | 在 rebuild-needed 中 | 保留在 rebuild-needed 中 |
| **rename方向** | 0061~0079→0082~0100(A2) | 0061~0079→0105~0123(A3) |
| **A2 rename** | 0061~0081→0082~0102 | 0080~0081→0101~0102 |

**关键改进：**
1. 引入 `material_id` 作为唯一真实归属标识
2. 不凭文件名编号猜测归属
3. 明确区分"文件存在但归属不同"与"文件缺失"两种场景
4. 重建的映射关系与队列元数据完全一致
