# 新需求记录：师生对话窗口与主动运营能力

> 来源：2026-03-31 与老师交流后汇总  
> 状态：**待讨论落地方案**，尚未并入正式开发计划  
> 说明：本文件仅作原始需求记录，不含技术结论，技术方案在后续专题文档中讨论。

---

## 一、需求概述

本次交流新增了四个方向，均涉及"师生对话"这一核心交互场景的升级。与原有方案中"学生提问-AI回答-教师处理"的异步分流模式不同，新需求希望更多地做到**同一个对话框里的多角色共存、主动触达、智能运营**。

---

## 二、需求详情

### 需求 1：抖音 AI 互动风格的对话界面

**原始描述：**  
> 师生对话窗口参考抖音 AI 互动的样式，同学跟 AI 导员说话，但真实导员也可以随时以自己身份插入对话。

**核心意图：**
- 界面风格去原型化，强调**沉浸感和拟人感**
- 学生感觉像在和一个"AI 版辅导员"对话，而不是使用一个表单系统
- AI 形象有人格化设计（可能包含头像、称谓、语气风格）

**关键交互模式——教师实时插入：**
- 真实教师可以"闯入"正在进行的对话
- 插入后，对话气泡需要明确标识"来自真实老师 × 老师"而不是 AI
- 学生能清楚感知这条消息的来源身份切换
- 教师插入后，AI 是否暂停、继续还是退出需要有明确策略

**参考产品理解（抖音 AI 互动风格）：**
- 气泡式对话流，但不只是纯文字——支持卡片、快捷指令、富媒体内容嵌入
- 有明显的 AI 标识（区分 AI 和真人身份）
- 支持快捷提问词/推荐问题，降低输入成本
- 情境感知型 UI（根据话题自动展示相关入口、链接、卡片）

---

### 需求 2：AI 主动发送消息（主动推送能力）

**原始描述：**  
> AI 可以主动给学生发送消息，比如最近学校有什么活动，导员希望 AI 主动提及并附带链接。

**核心意图：**
- AI 不只是"被动问答"，还具备**主动触达**能力
- 触达内容：学校活动通知、重要截止日期、事务办理提醒等
- 触达方式：在对话窗口主动发一条消息（不是短信或系统弹窗）
- 内容来源：由教师/管理员配置，而不是让 AI 自主生成

**隐含问题待讨论：**
- 主动消息的触发机制是什么？（定时？事件驱动？导员手动触发？）
- 面向对象是全体学生还是特定群体？
- 消息频率如何控制，避免打扰感？
- 消息是否进入会话流，还是单独的通知流？

---

### 需求 3：学生特征总结（学生画像能力）

**原始描述：**  
> 添加学生特征总结，方便老师对学生有了解。需要学生对话意愿强烈，不知道实践起来怎么样。

**核心意图：**
- 基于学生与 AI 的历史会话，自动或半自动生成该学生的"特征摘要"
- 教师在工作台可以查看某个学生的提问倾向、关注点、疑虑等
- 属于"辅导员对学生的数字化了解"的延伸

**现实约束（需明确）：**
- **数据来源**仅限系统内的会话记录，不接入外部数据源
- **学生参与意愿**是最大变量——画像依赖学生主动说话，若对话量少则无效
- **隐私问题**：学生画像属于敏感数据，需要明确告知、授权机制和访问权限控制
- **当前阶段判断**：这个功能对冷启动阶段（对话数据极少时）价值有限，建议在系统运行一段时间、积累一定对话数据后再引入

---

### 需求 4：定制导员——AI 申请导员介入 + 自动入库

**原始描述：**  
> 很多学生会批量提问时效性强但重复的问题，AI 此时可以申请导员介入，并自动入库，其他学生再问时直接复用。（称之为"定制导员"）

**核心意图：**
- 当 AI 判断某个问题"超出现有知识库 + 时效性强 + 重复度高"时，自动触发"请求导员介入"
- 导员处理后，这套问答自动生成知识草稿，经审核后进入知识库
- 后续其他同学再问同类问题，直接命中这条知识

**这个需求与现有设计的关系：**
- 本质是"未解决问题上报 → 教师处理 → 人工转知识库"这条闭环的**自动触发升级版**
- 原有设计中已经有这条链路，但是"人工判断上报"；这里增加的是"AI 主动判断触发"
- 新增的"自动入库"也只是把"教师答复转知识草稿"这一步做得更自动化

**关键技术问题待讨论：**
- AI 判断"需要介入"的置信度门槛如何设定？
- 同一问题被多个学生反复触发时，如何去重、合并处理？
- 导员介入时的 UI 是什么样的？（是在独立工作台处理，还是直接在该学生的对话框里出现？）

## 三、四个需求的关联性分析

这四个需求看似独立，但都指向同一个方向：**把对话窗口从"学生问/AI答"的单通道，升级为"AI 主动运营、教师可随时介入、知识自动增长"的智能服务场景**。

```
           ┌──────────────────────────────────────────┐
           │            师生对话窗口                    │
           │                                          │
           │  学生发起提问                             │
           │      ↓                                   │
           │  AI 回答（RAG + 人格化风格）              │
           │      ↓                                   │
           │  [主动推送] AI 也会主动发消息给学生        │
           │      ↓                                   │
           │  [定制导员] AI 判断超出范围 → 申请介入    │
           │      ↓                                   │
           │  [教师插入] 导员以真实身份进入对话        │
           │      ↓                                   │
           │  答复自动生成知识草稿 → 入库复用          │
           │      ↓                                   │
           │  [学生画像] 对话记录沉淀为特征摘要        │
           └──────────────────────────────────────────┘
```

## 五、待讨论的核心问题

在下一步落地方案讨论中，需要明确以下问题：

1. **对话界面的 UI 形态**：是全屏沉浸式对话？还是页面嵌入式？移动端和 Web 是否统一风格？
2. **教师插入后 AI 的行为**：暂停等待？退出对话？继续辅助？
3. **主动推送的运营机制**：谁来配置触发规则？学校维度还是班级维度？
4. **定制导员的触发逻辑**：如何判断"AI 处理不了"？基于置信度？还是关键词？还是人工标注触发词？
5. **学生画像的伦理边界**：是否需要学生知情同意？画像内容展示给谁？是否有访问权限分级？

## 六、需求 5：意图识别快捷链接推荐（2026-03-31 新增）

**原始描述：**
> 学生提问某个比赛如何报名、或报修设施等，系统自动弹出相关链接供学生快捷访问。

**核心意图：**
- AI 在理解学生提问意图后，在回答下方附带若干**快捷入口卡片**
- 卡片包含：链接标题 + 目标 URL，点击直接跳转
- 不替代 AI 文字回答，而是作为"行动入口"的补充

**与现有功能的关系：**
- 原有"事务导办模块"已规划了"链接聚合"能力，本需求是其**智能化版本**
- 原版是静态分类导航；这个需求是**动态意图匹配**——AI 根据提问内容自动推荐

**实现思路（低成本方案）：**

```
1. 管理员在后台维护一个"快捷链接库"
   └── 每条链接包含：名称、URL、描述、若干标签/关键词

2. AI 回答时，额外调用一次"链接匹配"逻辑
   └── 用问题 Embedding 与链接库 Embedding 做相似度匹配
   └── 找到相关度超过阈值的链接 → 附加到回答末尾

3. 前端展示为气泡下方的"快捷卡片"行
   └── 类似微信卡片式消息，点击打开外部链接
```

**数据结构（快捷链接条目）：**

| 字段 | 说明 |
|---|---|
| `name` | 链接名称，如"校外奖学金报名入口" |
| `url` | 目标链接 |
| `description` | 简短描述（用于向量化匹配） |
| `category` | 分类（比赛报名 / 设施报修 / 学籍办理 / …） |
| `tags` | 补充关键词 |
| `is_active` | 是否启用 |

**开发成本评估：低**
- 链接库 CRUD：若依后台直接生成，几乎不用写代码
- 向量匹配逻辑：复用现有 ChromaDB + DashScope Embedding，加一个独立集合
- 前端卡片组件：独立气泡卡片，设计稿风格一致，AI 一次性生成

**一期建议：** ✅ 纳入一期，与 AI 回答模块一起实现，成本极低但体验提升明显。
