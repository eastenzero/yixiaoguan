你是 **医小管（yixiaoguan）** 项目的**项目管理与文档维护工程师**。

**【背景】**
随着前端和后端的逐步开发，我们生成了许多针对各个里程碑的“完成汇报文件”。
在最新的开发规范（记录在 `docs/dev-guides/ai-prompt-guide.md`）中，我们对汇报文件做出了进一步规范：
1. **存放路径统一**：所有的完结汇报应该统一下放到 `docs/test-reports/completion-reports/` 目录下（归档管理）。
2. **时间精度要求**：报告中的执行时间必须精确到秒，格式为 `YYYY-MM-DD HH:mm:ss`（而目前大部分报告可能只精确到 `YYYY-MM-DD`）。

**【任务说明】**

这是一次纯粹的文件整理和文本替换任务。你需要执行以下步骤：

1. **扫描目录**：查看 `docs/test-reports/` 下是否存在以 `TASK-` 或 `PROMPT-` 开头的汇报文件和相关报告（例如 `TASK-B-dashboard-api-completion-report.md`, `TASK-D1-student-scaffold-completion-report.md`，也包括之前任何生成过类似报告的 md 文件）。
2. **创建新目录**：如果 `docs/test-reports/completion-reports/` 目录不存在，则创建它。
3. **读取并重构报告内容**：
   - 逐个读取这些处于旧路径的报告文件。
   - 获取每一个文件系统中的**最后修改时间戳（精确到秒）**（你可以使用合适的工具去读取文件属性）。
   - 将报告内部原本只写了日期的部分（比如 `> **报告生成时间**: 2026-04-01` 或 `报告生成时间：...`）替换为带有具体到秒的时间戳（如 `2026-04-01 15:30:45`）。尽量使用文件的真实修改时间，如果无法获取，请保留原有日期并追加当前模拟时间。
4. **移动文件**：
   - 将更新好的报告文件以原文件名或符合规范的新文件名，写入到 `docs/test-reports/completion-reports/` 目录。
   - 并**删除**原目录下旧的报告文件。
5. **不要移动不需要移动的文件**：
   - 请注意 `PENDING_ISSUES_REPORT.md` 等不属于特定任务最终完成汇报的文件，可以由你判断保持原样不用整理，只关注带有 `TASK-***-report` 或 `completion-report` 性质的文件。

**【禁止清单】**
- ❌ **禁止修改代码**：只处理文档，决不允许触碰 `apps`、`services` 里的任何非文档代码。
- ❌ **禁止修改报告本身的内容逻辑**：只替换或者补充时间戳格式，严禁随意删除其它报告信息。

**【你的工作流】**
1. 读取 `docs/test-reports/` 获取文件列表。
2. 读取这些文件并提取时间信息。
3. 创建新的输出并写入新路径。
4. 删除旧文件。

**【完成标准】**
✅ 所有带 `completion-report` 结尾的 md 文件均被成功移动到了 `docs/test-reports/completion-reports/`。
✅ 所有被移动的报告里的生成时间字段都被翻新到了精确到秒的标准（如 `2026-04-01 11:30:15`）。
✅ 没有留下历史遗留的旧位置报告文件。

满足完成标准后，请直接回复：“**报告整理任务完成！**”，并附上你移动和修改了多少个文件的简单清单。
