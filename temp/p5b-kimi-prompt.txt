## 修复二：多轮对话 history 传递（DEBT-04）

当前问题：handleChatIntent 第89行 new ChatStreamRequest(query) 没有传 history，每轮对话无上下文。

修复步骤：

1. 先读 ChatStreamRequest.java，确认 history 字段类型（通常是 List<Map<String,String>> 或 List<HistoryItem>），以及 role 字段的值（"user"/"assistant" 还是其他）。

2. 先读 IYxMessageService.java，找查询历史消息的方法（可能叫 getMessagesByConversationId、listMessages 等），确认方法签名。

3. 在 handleChatIntent 中，构造 ChatStreamRequest 之前，查询最近 6 条消息作为 history：

示意代码（需根据实际接口调整）：
```java
// 查询最近6条历史消息（不含当前这条）
List<YxMessage> recentMessages = messageService.getRecentMessages(conversationId, 6);

// 构造 history（格式参照 ChatStreamRequest.history 的元素类型）
List<Map<String, String>> history = new ArrayList<>();
for (YxMessage msg : recentMessages) {
    Map<String, String> item = new HashMap<>();
    item.put("role", msg.getSenderType() == 1 ? "user" : "assistant");
    item.put("content", msg.getContent() != null ? msg.getContent() : "");
    history.add(item);
}

// 构造请求，传入 history
ChatStreamRequest request = new ChatStreamRequest(query);
request.setHistory(history);
```

注意：
- history 中必须只包含有实际内容的消息（过滤 content 为空的）
- 如果 ChatStreamRequest 构造器已支持 history 参数，直接用；否则用 setter
- role 字段的值必须与 ai-service 接口定义一致（读 ChatStreamRequest.java 确认）
