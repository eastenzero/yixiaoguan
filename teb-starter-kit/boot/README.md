# Boot Prompts（身份卡）

每个文件是一张 AI 角色的"身份卡"，约 200 token。

## 用法

新开 AI 对话时，把对应角色的 `.md` 文件拖进对话窗口（或复制粘贴内容），AI 即刻进入该角色。

| 文件 | 角色 | 何时用 |
|------|------|--------|
| `t0.md` | 架构师 | 需求澄清、架构设计、Bug 诊断 |
| `t1.md` | 协调者 | 任务分解、batch 编排、L3 验收 |
| `t2.md` | 包工头 | 任务分发、L0-L2 验证、git commit |
| `t3-executor.md` | 执行者 | 写代码、改文件、跑自检 |
| `t3-scout.md` | 侦察员 | 只读信息收集、结构化摘要 |

## 为什么不直接用完整提示词？

完整提示词（`prompts/` 目录）每个 2-5K token，全部加载会占用 10-20K 上下文。身份卡压缩到 ~200 token，只包含角色红线和核心职责，对于大多数场景足够。

AI 如果需要查阅详细规则，身份卡末尾有指向完整提示词的路径引用。
